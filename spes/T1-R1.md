# T1-R1 — Dynamic Commission Rules Summary

## Data Analysis

### Primary Table: `commission_rules` (or derived from `rules_configurations`)

| Field | Type | Description |
|-------|------|-------------|
| course_id | UUID, FK | Links to courses table |
| rule_type | VARCHAR | "PERCENTAGE" or "FLAT" |
| rule_value | DECIMAL | Commission value (e.g., 15.0 for 15%) |
| effective_date | DATE | When this rule becomes active |
| expiry_date | DATE | When this rule expires |

### Relationships
- Links to `courses` table via `course_id` → provides course name, stream, duration

## UI Design Guidelines

### Layout
- Sortable table with filter by course/stream and date range picker (active rules only)
- Status badges for active/inactive/expired states

### Filters
- Course dropdown (from courses table)
- Stream filter (e.g., Science/Arts/Commerce)
- Date range: from effective_date to expiry_date

## API Development Guidelines

### Endpoint: `/api/reports/commission-rules`

**Request Schema:**
```json
{
  "course_id": "uuid or null",
  "stream": "string or null",
  "from_date": "YYYY-MM-DD",
  "to_date": "YYYY-MM-DD"
}
```

**Response Schema (200 OK):**
```json
[
  {
    "course_name": "B.Tech Computer Science",
    "stream": "Science",
    "commission_type": "PERCENTAGE",
    "value": 15.0,
    "effective_date": "2024-01-01",
    "expiry_date": "2026-12-31"
  }
]
```

### Authentication
- Uses existing SPES JWT auth (same as other endpoints)

## Output Format Specifications

### CSV Export Headers:
`Course Name | Stream | Commission Type | Value | Effective Date | Expiry Date`

### Excel Export:
- Same columns, with formatted date cells and currency formatting for value column
