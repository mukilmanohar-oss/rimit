# T1-R5 — University Performance Overview

## Data Analysis

### Primary Tables: `universities`, `courses`, `enrollments`, `payment_ledgers`

| Field | Type | Description |
|-------|------|-------------|
| id | UUID, PK | Unique university identifier |
| name | VARCHAR, UNIQUE | University name |
| state | VARCHAR | Geographic location |
| accreditation | VARCHAR | Accreditation body |

### Aggregations per University
- Total courses offered
- Number of enrolled students
- Revenue generated from payments

## UI Design Guidelines

### Layout
- Dashboard-style cards with KPIs (total courses, enrolled students, revenue)
- Table listing universities sorted by performance metrics

### Filters
- State dropdown or search box
- Date range for enrollment/payment data

## API Development Guidelines

### Endpoint: `/api/reports/university-performance`

**Request Schema:**
```json
{
  "state": "string or null",
  "from_date": "YYYY-MM-DD",
  "to_date": "YYYY-MM-DD"
}
```

**Response Schema (200 OK):**
```json
[
  {
    "university_name": "RIMIT University",
    "state": "Telangana, India",
    "accreditation": "NAAC A+",
    "total_courses": 45,
    "enrolled_students": 1200,
    "revenue_generated": 850000.00
  }
]
```

### Authentication
- Uses existing SPES JWT auth (same as other endpoints)

## Output Format Specifications

### CSV Export Headers:
`University Name | State | Accreditation | Total Courses | Enrolled Students | Revenue Generated`

### Excel Export:
- Same columns, with formatted date cells and currency formatting for revenue column
