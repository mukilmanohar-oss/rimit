# T1-R3 — Commission Breakdown Report

## Data Analysis

### Primary Tables: `payment_ledgers`, `enrollments`, `courses`, `fee_structures`

| Field | Type | Description |
|-------|------|-------------|
| course_id | UUID, FK | Links to courses table |
| fee_type | VARCHAR | "PERCENTAGE" or "FLAT" |
| amount | DECIMAL | Fee amount for this structure |

### Calculations
- Commission per transaction based on rule matching (from commission_rules)
- Total commission breakdown by course/stream hierarchy

## UI Design Guidelines

### Layout
- Multi-level table with course/stream hierarchy and individual transactions listed below
- Expandable rows to show detailed transaction data

### Filters
- Course dropdown, stream filter, date range for payments

## API Development Guidelines

### Endpoint: `/api/reports/commission-breakdown`

**Request Schema:**
```json
{
  "course_id": "uuid or null",
  "stream": "string or null",
  "from_date": "YYYY-MM-DD",
  "to_date": "YYYY-MM-DD"
}
```

**Response Schema (200 OK):**
```json
[
  {
    "course_name": "B.Tech Computer Science",
    "stream": "Science",
    "fee_type": "PERCENTAGE",
    "amount_paid": 15000.00,
    "commission_value": 2250.00,
    "transaction_ref": "TXN-2026-78945"
  }
]
```

### Authentication
- Uses existing SPES JWT auth (same as other endpoints)

## Output Format Specifications

### CSV Export Headers:
`Course Name | Stream | Fee Type | Amount Paid | Commission % | Commission Value | Transaction Ref | Date`

### Excel Export:
- Same columns, with formatted date cells and currency formatting for monetary values
