# T1-R8 — Sub-Center Performance Scorecard

## Data Analysis

### Primary Tables: `sub_centers`, `enrollments`, `payment_ledgers`

| Field | Type | Description |
|-------|------|-------------|
| center_code | VARCHAR, UNIQUE | Identifier for the sub-center |
| location | VARCHAR | Physical address of the center |
| status | VARCHAR | "ACTIVE", "INACTIVE", or "UNDER_REVIEW" |

### Aggregations per Sub-Center
- Enrollment count (from enrollments table)
- Revenue generated (sum of payment_ledgers.amount_paid)
- Average transaction value

## UI Design Guidelines

### Layout
- Ranked table with scorecard-style metrics (KPIs per center)
- Visual indicators for performance tiers (e.g., color-coded rows or badges)

### Filters
- Status dropdown (ACTIVE/INACTIVE/UNDER_REVIEW)
- Location search box

## API Development Guidelines

### Endpoint: `/api/reports/sub-center-scorecard`

**Request Schema:**
```json
{
  "status": "string or null",
  "location": "string or null"
}
```

**Response Schema (200 OK):**
```json
[
  {
    "center_code": "RIMIT-HYD-01",
    "location": "Hyderabad, India",
    "status": "ACTIVE",
    "enrollments": 45,
    "revenue": 22500.00,
    "avg_transaction_value": 500.00,
    "performance_tier": "HIGH"
  }
]
```

### Authentication
- Uses existing SPES JWT auth (same as other endpoints)

## Output Format Specifications

### CSV Export Headers:
`Center Code | Location | Status | Enrollments | Revenue | Avg Transaction Value | Performance Tier`

### Excel Export:
- Same columns, with formatted date cells and currency formatting for monetary values
