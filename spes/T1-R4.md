# T1-R4 ‚Äî Student Enrollment Payment Gateway Transaction Log

## Data Analysis

### Primary Tables: `enrollments`, `payment_ledgers`, `students`

| Field | Type | Description |
|-------|------|-------------|
| student_id | UUID, FK | Links to students table |
| full_name | VARCHAR | Student's full name |
| email | VARCHAR, UNIQUE | Student email address |
| primary_phone | VARCHAR | Primary phone number |

### Relationships
- `enrollments` ‚Üí `students` (student details)
- `payment_ledgers` ‚Üí `enrollments` (payment records for each enrollment)

## UI Design Guidelines

### Layout
- Chronological table with student details and payment metadata
- Filter by date range or status (paid/pending/failed)

### Filters
- Date range picker (from_date, to_date)
- Status dropdown (all/Paid/Pending/Failed)

## API Development Guidelines

### Endpoint: `/api/reports/student-payment-log`

**Request Schema:**
```json
{
  "student_id": "uuid or null",
  "enrollment_id": "uuid or null",
  "from_date": "YYYY-MM-DD",
  "to_date": "YYYY-MM-DD"
}
```

**Response Schema (200 OK):**
```json
[
  {
    "student_name": "Rahul Kumar",
    "email": "rahul@example.com",
    "phone": "+91-9876543210",
    "course": "B.Tech Computer Science",
    "session": "Fall 2025",
    "amount_paid": 12500.00,
    "transaction_ref": "PG-TXN-987654",
    "gateway_status": "SUCCESS"
  }
]
```

### Authentication
- Uses existing SPES JWT auth (same as other endpoints)

## Output Format Specifications

### CSV Export Headers:
`Student Name | Email | Phone | Course | Session | Amount Paid | Transaction Ref | Gateway Status | Timestamp`

### Excel Export:
- Same columns, with formatted date cells and currency formatting for monetary values
