# T1-R6 — Admission Funnel Analytics

## Data Analysis

### Primary Tables: `intake_sessions`, `enrollments` (with status), `students`

| Field | Type | Description |
|-------|------|-------------|
| session_name | VARCHAR | Name of the intake session |
| start_date | DATE | Session start date |
| is_active | BOOLEAN | Whether session is currently active |

### Funnel Stages (tracked via enrollment status)
- Applied → Enrolled → Paid → Active
- Conversion rates per stage calculated from counts

## UI Design Guidelines

### Layout
- Visual funnel diagram with percentage drop-offs between stages
- Table below showing counts and conversion metrics

### Filters
- Session dropdown or search box
- Date range for session data

## API Development Guidelines

### Endpoint: `/api/reports/admission-funnel`

**Request Schema:**
```json
{
  "session_name": "string or null",
  "from_date": "YYYY-MM-DD",
  "to_date": "YYYY-MM-DD"
}
```

**Response Schema (200 OK):**
```json
[
  {
    "session_name": "Fall 2025",
    "start_date": "2025-09-01",
    "applied": 450,
    "enrolled": 380,
    "paid": 365,
    "active": 340,
    "conversion_rates": {
      "applied_to_enrolled": 84.4,
      "enrolled_to_paid": 96.1,
      "paid_to_active": 93.2
    }
  }
]
```

### Authentication
- Uses existing SPES JWT auth (same as other endpoints)

## Output Format Specifications

### CSV Export Headers:
`Session Name | Start Date | Applied | Enrolled | Paid | Active | Conversion Rates (JSON or separate columns)`

### Excel Export:
- Same columns, with formatted date cells and percentage formatting for conversion rates
