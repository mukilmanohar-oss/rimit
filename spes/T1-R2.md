# T1-R2 — Center-wise Ledger Summary

## Data Analysis

### Primary Table: `payment_ledgers` joined with `sub_centers`, `enrollments`, `students`

| Field | Type | Description |
|-------|------|-------------|
| center_code | VARCHAR, FK | Links to sub_centers table |
| location | VARCHAR | Physical address of the center |
| amount_paid | DECIMAL | Total payment amount |
| transaction_ref | VARCHAR, UNIQUE | Payment gateway reference ID |
| status | VARCHAR | "PAID", "PENDING", "FAILED" |

### Aggregations
- Sum of `amount_paid` per center
- Count of transactions by status (paid/pending/failed)

## UI Design Guidelines

### Layout
- Table grouped by center with status badges and totals at bottom
- Color-coded rows: green for paid, yellow for pending, red for failed

### Filters
- Center dropdown or search box
- Date range picker for transaction dates

## API Development Guidelines

### Endpoint: `/api/reports/center-ledger`

**Request Schema:**
```json
{
  "center_code": "string or null",
  "from_date": "YYYY-MM-DD",
  "to_date": "YYYY-MM-DD"
}
```

**Response Schema (200 OK):**
```json
[
  {
    "center_code": "RIMIT-HYD-01",
    "location": "Hyderabad, India",
    "total_paid": 45000.00,
    "pending_amount": 3200.00,
    "failed_transactions": 1,
    "last_updated": "2026-07-14T10:30:00Z"
  }
]
```

### Authentication
- Uses existing SPES JWT auth (same as other endpoints)

## Output Format Specifications

### CSV Export Headers:
`Center Code | Location | Total Paid | Pending Amount | Failed Transactions | Last Updated`

### Excel Export:
- Same columns, with formatted date cells and currency formatting for monetary values
