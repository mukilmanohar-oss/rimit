# T1-R7 — Course Revenue & Enrollment Summary

## Data Analysis

### Primary Tables: `courses`, `fee_structures`, `enrollments`, `payment_ledgers`

| Field | Type | Description |
|-------|------|-------------|
| name | VARCHAR | Course name |
| stream | VARCHAR | Academic stream (Science/Arts/etc.) |
| duration_months | INT | Program length |
| is_active | BOOLEAN | Whether course is currently active |

### Aggregations per Course
- Total enrollments count
- Unique students enrolled
- Revenue from fee structures

## UI Design Guidelines

### Layout
- Table with course details and aggregated metrics
- Filter by active/inactive status

### Filters
- Stream dropdown (Science/Arts/Commerce/etc.)
- Active status toggle

## API Development Guidelines

### Endpoint: `/api/reports/course-revenue`

**Request Schema:**
```json
{
  "stream": "string or null",
  "is_active": true|false|null,
  "from_date": "YYYY-MM-DD",
  "to_date": "YYYY-MM-DD"
}
```

**Response Schema (200 OK):**
```json
[
  {
    "course_name": "B.Tech Computer Science",
    "stream": "Science",
    "duration_months": 4,
    "total_enrollments": 156,
    "unique_students": 142,
    "revenue": 78000.00
  }
]
```

### Authentication
- Uses existing SPES JWT auth (same as other endpoints)

## Output Format Specifications

### CSV Export Headers:
`Course Name | Stream | Duration (months) | Total Enrollments | Unique Students | Revenue`

### Excel Export:
- Same columns, with formatted date cells and currency formatting for revenue column
